Usando o GIT/GitHub

  • Para ver lista de arquivos: ls
  • Para limpar tela: ctrl + l
  • Voltar um nível: cd..
  • Completar uma palavra: tecla TAB
  • Percorrer ítens seta para cima do teclado
  1. Clica com o botão direito no local que deseja abrir o git e clica em: git bash here

  2. Para entrar numa pasta: cd nome da pasta
    3. Exemplo: Se a pasta se chama workspace: cd workspace

  3. Para criar uma pasta: mkdir nome da pasta a ser criada
    4. Exemplo: Se quero criar uma pasta chamada livro-receitas: mkdir livro-receitas

  4. Rodar o git unit para o git versionar o código: git unit

  5. Flag para mostrar arquivos ocultos, como por exemplo a pasta ".git"(pasta gerencial do git): ls -a

  6. Entrar na pasta ".git": cd .git/

  7. Adicionar arquivos para serem commitados: git add .

  8. Criando commit mais mensagem: git commit -m "mensagem a ser inserida" (mensagem entre aspas)

  9. Ver o status do arquivo: git status

  10. Mover arquivo para um repositório: mv strogonoff.md ./receitas/
    11. Exemplo: Se eu tiver um arquivo de nome "strogonoff.md" e quero mover para a pasta(repositório) "receitas"

  11. Verificar as configurações do git: git config --list

  12. Enviar arquivo local p/o remoto(git hub): git remote add origin (link obtido do git hub)

  13. Ver listas de repositórios remotos cadastradas: git remote -v

  14. "Empurrar" o repositório local para o remoto: git push origin master

Observações:

Criar os comandos para configuração de dados do autor
- Para email: git config --global user.email "email desejado" (entre aspas)
- Para o username: git config --global user.name "username desejado"(sem as aspas)

Alterar os dados da configuração do autor
- Para email: git config --global --unset user.email
- Para username: git config --global --unset user.name

Abrir o VSCode no GitHub
- Na página do repositório clicar em: . (tecla ponto)

Mais alguns comandos

git restore --source (numero do commit) . - Usado para voltar um commit. O ponto indica que todos os arquivos sofrerão modificação, podemos colocar o nome do arquivo indivialmente mais a sua extensão indicando onde queremos modificar.

git switch (nome da branch) - Para trocar de branch - similar ao git checkout (nome da branch)

git merge (nome da branch que deseja mergear) - Fazer merge localmente

git log --oneline - Ver o log da nossa aplicação em apenas uma linha.
