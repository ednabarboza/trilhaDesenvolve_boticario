HTTP: Entendendo a web por baixo dos panos

HTTP(Hypertext Transfer Protocol): é um Protocolo de Transferência de Hipertexto que propõe regras para a comunicação entre cliente e servidor. Mas atenção, quando usamos esse protocolo todos os dados enviados entre cliente e servidor são transmitidos em texto puro, inclusive dados sensíveis, como login e senha! O HTTP é stateless (não mantém informações e cada requisição feita é única). Usa sessões para guardar informações nos cookies.

No mundo HTTP a comunicação segue sempre esse modelo: o cliente envia uma requisição e o servidor responde. Requisição e Resposta ou em inglês: Request-Response. Aqui é importante saber que a comunicação sempre começa com o cliente: é ele quem pede as informações. O servidor responde apenas o que foi requisitado e nunca inicia a comunicação!

*Sessão: é o tempo que o cliente permanece ativo no sistema

*Cookies: pequeno arquivo de texto que guarda informações sobre o usuário no navegador (fica associado com o domínio e guarda o número de session ID).

HTTPS - A versão segura do HTTP

HTTPS: é o HTTP comum, porém com uma camada adicional de segurança/criptografia (SSL/TLS). O HTTPS para garantir segurança usa criptografia baseada em chaves públicas e privadas e para gerar essas chaves públicas e privadas é preciso garantir a identidade de quem possui essas chaves e isso é feito a partir de um certificado digital, ou seja, um certificado digital é utilizado para identificar determinada entidade e ainda é utilizada para geração das chaves de criptografia. Apesar disso, ainda é necessário que uma autoridade certificadora, que nada mais é que um órgão ou entidade confiável, garanta não apenas a identidade do site mas também a validade do certificado.

*SSL (Secure Sockets Layer) e TLS (Transport Layer Security)

Criptografia Simétrica

HTTPS usa ambos os métodos de criptografia, assimétrica e simétrica. Ele começa com criptografia assimétrica e depois muda para criptografia. simétrica

DNS (Domain Name System): realiza a tradução do nome de um domínio para o endereço ID correspondente

Portas

Por padrão a porta 80 é referente ao método HTTP e a porta 443 ao HTTPS.

Métodos HTTP

Status Code

O status code do HTTP tem relação com o que aconteceu no processamento do pedido

Para fazer requisições usando métodos como PUT, PATCH e DELETE precisamos utilizar códigos JavaScript. Esses métodos são mais utilizados em web services

Parâmetros na requisição

Nas requições realizadas podemos especificar o formato que esperamos receber a resposta, para isso usamos o cabeçalho accept

Exemplo: Accept: text/html, Accept: text/css, Accept: application/xml, Accept: application/json, etc

REST (Representation State Transfer)

  • É um padrão arquitetal para comunicação entre aplicações.
  • Aproveita da arquitetura do HTTP
  • Recursos são definidos via URI
  • Operações com os métodos HTTP (GET, POST, PUT, DELETE)
  • Cabeçalhos (Accept/Content-Type) para especificar a representação (JSON, XML, etc)

HTTP/2

HTTP/2: traz os dados em binário, comprimidos num algoritmo chamado hpack e criptografados com TLS. O cabeçalho é stateful (não guarda estado)

*hpack é usado apenas para headers

Server Push no HTT/2

O servidor pode empurrar para os clientes certos recursos antes mesmo de serem solicitados, pois ele consegue analisar o HTML e ver o que é preciso para carregar a página

HTTP/2 é Multiplexing

Requisição e respostas são paralelas

HTTP/2 é Headers Stateful

São enviados apenas os cabeçalhos que tem mudanças
